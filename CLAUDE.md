# CLAUDE.md — EPI-DevorApp

## Project overview

Restaurant discovery and recommendation app. Users can browse restaurants, call them, rate them, and receive personalised recommendations via a Keras neural network. The web frontend is also packaged as an Android APK distributed through Google Play.

## Repository layout

```
epi-devorapp/
├── frontend/              # React 19 + TS + Vite web app, Capacitor Android wrapper
│   ├── src/
│   │   ├── components/    # Shared UI components
│   │   ├── controllers/   # Business logic / API calls
│   │   ├── models/        # TypeScript data models
│   │   ├── views/         # Page-level components
│   │   └── tests/         # Vitest unit tests
│   ├── android/           # Generated by `npx cap add android` — commit this folder
│   ├── fastlane/          # Fastlane lanes for Play Store deployment

│   ├── vite.config.ts     # Proxy /api → localhost:8000, HTTPS via BasicSSL
│   └── Gemfile            # Locks fastlane version
├── backend/               # FastAPI + SQLAlchemy + Alembic
│   ├── app/
│   │   ├── core/          # Config, settings
│   │   ├── data/          # DB session, repositories
│   │   ├── models/        # SQLAlchemy ORM models
│   │   ├── presentation/  # FastAPI routers
│   │   └── services/      # Business logic
│   ├── alembic/           # DB migration scripts
│   └── tests/             # pytest + pytest-asyncio

```

## Key commands

### Frontend

```bash
cd frontend
npm install          # install dependencies (includes Capacitor)
npm run dev          # dev server at https://localhost:5173
npm run build        # production Vite build → dist/
npm run test         # vitest unit tests
npm run lint         # eslint
npm run cap:sync     # sync dist/ into android project
npm run cap:open     # open Android Studio
npm run cap:build    # build + sync in one step
```

### Backend

```bash
cd backend
poetry install --with dev
poetry run uvicorn app.main:app --reload   # dev server :8000
poetry run pytest tests/ -v               # run tests
poetry run alembic upgrade head            # apply DB migrations
```

### Docker (full stack)

```bash
docker compose up --build    # backend :8080, frontend :5173
```

## CI/CD pipeline (.gitlab-ci.yml)

| Stage | Job | Runs on | Notes |
|---|---|---|---|
| test | `test_backend` | main, MR, tags | pytest |
| test | `lint_frontend` | main, MR, tags | eslint |
| test | `test_frontend` | main, MR, tags | vitest |
| build | `build_backend` | main, tags | Docker image → registry |
| build | `build_frontend` | main, tags | Docker image → registry |
| android | `build_apk` | main, tags | Needs test_frontend + lint_frontend only |
| deploy | `deploy_play_store` | tags only | Uploads to Play Store internal track |

A tagged push (`git tag v1.x.x && git push origin v1.x.x`) triggers the full pipeline including Play Store upload.

## Android setup (one-time, local)

The `frontend/android/` folder must exist in the repo. If it's missing:

```bash
cd frontend
npx cap add android     # generates android/
npx cap sync android    # copies dist/ into android
git add android/
git commit -m "Add Capacitor Android project"
```

App ID: `com.devorapp.epi` (set in `capacitor.config.ts` and `fastlane/Appfile`)

| `KEYSTORE_KEY_ALIAS` | variable | Key alias |
| `KEYSTORE_KEY_PASSWORD` | masked | Key password |

## Important notes

- `frontend/android/` must be committed. The CI does **not** run `cap add android`; it only runs `cap sync`.
- The `deploy_play_store` job uploads to the **internal** track. Promote to production via Play Console or change `track:` in `frontend/fastlane/Fastfile`.
- The Android SDK command-line tools URL in `.gitlab-ci.yml` (`commandlinetools-linux-11076708_latest.zip`) may need updating — get the latest from https://developer.android.com/studio#command-line-tools-only.
- Vite dev server uses HTTPS (BasicSSL plugin). The API proxy (`/api → localhost:8000`) is only active in dev mode; production builds talk directly to the deployed backend URL.
- Firebase credentials must not be committed; they are loaded via environment variables / service account files listed in `.gitignore`.
